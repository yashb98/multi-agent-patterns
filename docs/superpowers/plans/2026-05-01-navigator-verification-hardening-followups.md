## 2026-05-01 Pipeline correctness fixes — applied

The follow-up plan `2026-05-01-pipeline-correctness-fixes.md` shipped 5 fixes on branch `pipeline-correctness-fixes`:

1. ✅ **dry_run plumbing through `route_and_apply`** (commit 71d67fa) — `process_single_url(dry_run=True)` no longer silently drops at the routing seam. `dry_run` is now a keyword-only argument with `True` default. `job_autopilot.py` confirmed to have no `route_and_apply` callers (cron path uses `_queue_for_review` directly — draft-only mode).
2. ✅ **`agent_rules` domain normalization** (commit b434f39) — `_normalize_domain` helper applied at both `auto_generate_from_correction` and `get_field_overrides`. Migration runs on `_init_db` for any rule with `rule_type='correction_override'` or correction-style sources. **Discovery during execution:** the 7 production rules turned out to be regex patterns (e.g. `programming.*language|backend.*language`), not domain strings — so the migration ran but found nothing to normalize. Fix is still valuable for future rules generated by `auto_generate_from_correction`.
3. ✅ **`job_analytics` DB path** (commit d82f7b1) — `_DB_PATH` points at `applications.db` instead of `jobpulse.db`. **Real impact:** `get_conversion_funnel` now returns `{applied: 9, rejected: 213, skipped: 188}` vs previously always 0. Adaptive Gate 3 thresholds will now actually adapt.
4. ✅ **Failure learning into MemoryManager** (commit 3098596) — `_record_failure_episode` calls `MemoryManager.record_episode` for failed strategies, called alongside `_feed_experience_memory` in `reflect_on_application`. Implementer used `get_shared_memory_manager()` singleton instead of bare `MemoryManager()` for connection cost — good call.
5. ✅ **Cleanup: 4 ghost DBs deleted, `determine_match_tier` deduplicated** (commit 32450ca) — **Plan adjusted at execution time:** spared 2 of the 6 originally-targeted ghost DBs because they have live writers (`gate_thresholds.db` from `GateThresholdAdapter._init_db()`, `project_selection_outcomes.db` from `ProjectOutcomeTracker._init_db()`). They're 0 bytes only because the classes haven't been instantiated yet. Deleting would have caused FileNotFoundError on first instantiation. Deleted: `jobs.db`, `navigation_sequences.db`, `page_reasoner_cache.db`, `profile.db`. Total `data/*.db` count: 55 → 51.

**Test additions:** 17 new tests across 4 files, all passing.

## Remaining followups (not in scope of pipeline-correctness-fixes plan)

These came up during the audit but were deferred:

- **Cron auto-submit dry-run-first refactor.** The cron path goes through `_queue_for_review` directly today (draft-only mode), so the original audit concern about cron bypassing dry-run+confirm may be partially moot. Verify whether any other cron path actually auto-submits, and if so, route through `apply_job(dry_run=True)` + `confirm_application()` properly.
- **Pre-submit gate revival.** `PreSubmitGate` only fires when `company_research is not None` (`__init__.py:235`), and `live_review_applicator` never passes it. Either remove the dead gate or wire it.
- **Mutex coverage of fill/submit.** `_apply_lock` only protects quota recording, not the actual fill/submit. Concurrent `apply_job()` calls can interleave their browser sessions.
- **Two parallel verification systems.** `NativeFormFiller` has its own per-field verification (`_fill_by_label:879-922`); the new `ExecutorResult` from the navigator-verification-hardening branch lives only on the navigator+auth path. They never reconcile — corrections from one path never teach the other's DBs.
- **AgentRulesDB orphan rules root cause.** The 7 production rules with `times_applied=0` are regex patterns from `user_correction`/`user_feedback` sources, not domain strings — the audit's domain-mismatch hypothesis was wrong. Trace where these rules originate and whether the consume path (`get_field_overrides`) is even the right reader for them. May be a separate dead-loop than the one Task 2 fixed.
- **`gate_effectiveness` table.** Schema exists; never receives a row. Trace why the writer is unreachable.
- **`draft_applicator.py` (~900 lines)** — fully dead code. Either delete or wire.
- **`GateThresholdAdapter` and `ProjectOutcomeTracker`** — both fully implemented with proper schemas and writer functions, but never instantiated in production. Decide whether to wire or remove.
- **`ai_assist_logger.ai_fixes_count` always 0** — verify whether external AI fixes are actually being logged.
- **`scan_pipeline.py` side-effect changes in commit 71d67fa** — CV PDF generation moved into `generate_materials` (changes the cost model — every scanned job now triggers PDF generation), and `cl_drive_link` hardcoded to `None` at two call sites. These were pre-existing uncommitted user changes that got swept in. Audit whether they're intentional.
